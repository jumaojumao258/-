## 背景与动机

在外贸与跨境业务中，汇率波动直接影响企业利润与资金安全。传统人工盯盘模式存在以下痛点：

- **反应滞后**：从汇率异动到人工决策之间存在分钟级甚至小时级延迟
- **信息碎片化**：汇率波动往往由宏观经济事件、央行政策、地缘政治等多因素叠加驱动，人工难以实时关联
- **决策非结构化**：缺乏标准化的风控 SOP，依赖个人经验，难以沉淀和复现

本项目旨在构建一个 **7×24 小时自动化汇率监控与决策辅助系统**，通过多 Agent 协作将「感知 → 分析 → 决策」链路压缩至秒级。

## 系统架构

系统采用 **三 Agent 流水线架构**，各 Agent 职责明确、数据流单向递进：

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│  数据感知 Agent  │ ──→ │  资讯推理 Agent  │ ──→ │  风控决策 Agent  │
│  Perception      │     │  Reasoning       │     │  Decision       │
├─────────────────┤     ├─────────────────┤     ├─────────────────┤
│ • 多币种高频拉取  │     │ • 异常触发检索    │     │ • 结构化研报生成  │
│ • 实时波动率计算  │     │ • 全球新闻提取    │     │ • 对冲策略建议    │
│ • 告警阈值比对   │     │ • 情绪分析       │     │ • 企业微信/飞书  │
│ • 7×24 持续监控  │     │ • 事件驱动因子    │     │   多渠道推送     │
└─────────────────┘     └─────────────────┘     └─────────────────┘
         │                       │                       │
         ▼                       ▼                       ▼
    数据层                信息层                   决策层
    高频行情             宏观经济新闻             风控行动
```

### 1. 数据感知 Agent

高频拉取多币种（USD/CNY、EUR/CNY、JPY/CNY、AUD/CNY、GBP/CNY）实时汇率，持续计算波动率并与预设阈值比对，一旦触发立即唤醒下游 Agent。

- ⏱ 更新频率：秒级（可配置）
- 📊 波动维度：实时价差、滚动窗口波动率、异常偏离度

### 2. 资讯推理 Agent

当汇率异动被触发后，该 Agent 自动检索全球宏观经济新闻，提取事件驱动因子并执行情绪分析，将原始行情信号转化为可理解的归因结论。

- 🔍 信息来源：对接新闻 API / RSS 聚合
- 🧠 推理引擎：基于大模型的因果分析与情绪打分

### 3. 风控决策 Agent

综合行情与资讯研判，生成结构化风控研报，包含短期走势预测与具体对冲操作建议，并通过企业微信/飞书等渠道实时推送。

- 📋 输出格式：结构化研报（原因分析 + 趋势预测 + 操作建议）
- 🔔 推送渠道：企业微信、飞书、邮件（可扩展）

## 功能特性

- **🌐 多币种实时监控**：USD/CNY、EUR/CNY、JPY/CNY、AUD/CNY、GBP/CNY 五大主流货币对
- **⚡ 异常波动秒级告警**：超过波动阈值自动触发报警
- **🧠 AI 驱动智能研判**：结合大模型进行波动原因追溯、短期趋势预测、操作建议生成
- **📊 可视化波动面板**：波动率进度条、多维数据卡片一目了然

- **🌙 深色主题 UI**：护眼且科技感十足的视觉风格

## 快速开始

### 在线体验

直接打开 [realtime-exchange-agent-sample.html](./realtime-exchange-agent-sample.html) 即可看到模拟运行的汇率监控面板，页面内置了模拟数据刷新逻辑（3.2 秒/次）。

### 本地运行

```bash
# 克隆仓库
git clone https://github.com/jumaojumao258/-.git

# 直接打开 HTML 文件
open realtime-exchange-agent-sample.html
# 或用浏览器直接打开
```

### 对接后端

如需对接真实数据源与 LLM 推理服务，可参考以下接口约定：

```javascript
// 感知层接口（GET）
GET /api/v1/rates
→ { usd_cny: 6.95, eur_cny: 7.63, ... }

// 推理层接口（POST）
POST /api/v1/analyze
Body: { alerts: [...] }
→ { reason, trend, action }

// 推送层接口（POST）
POST /api/v1/push
Body: { channel: "wechat"|"feishu", message: {...} }
```

## 效果预览

页面包含两个核心面板：

**左面板 — 当前汇率状态**

- 波动原因分析（LLM 推理结果）
- 短期趋势预测
- 结售汇操作建议
- 三 Agent 工作流说明
- 团队规模 / 响应延迟 / Token 消耗等运营数据
